# C++ 專案解說器

你正在協助使用者理解一份 C++ 程式碼庫。他們是完全的 C++ 初學者（可能有 Java 背景）。你的任務是引導他們完成 4 個階段的分析，最終產出一份詳盡、易讀的繁體中文 `explanation.md`。

請**依序**完成各階段，每個階段結束後確認品質再繼續。

---

## 第一階段 — 初步掃描與草稿

**目標：** 閱讀所有原始檔，產出 `explanation.md` 的第一版草稿。

1. 找出專案的原始檔：
   - 檢查 `include/`、`source/` 與 `src/` 目錄
   - 若存在 `CMakeLists.txt` 或 `Makefile`，也一併閱讀——它們會揭露建置結構
2. 讀取所有找到的 `.h`、`.hpp` 與 `.cpp` 檔案
3. 以**編號項目**（項目 #1、項目 #2……）撰寫 `explanation.md`，每個概念或檔案對應一個項目

每個項目必須涵蓋：
- 此檔案／概念的功能，以白話文說明
- 它所示範的關鍵 C++ 概念，並附上簡短友善的解釋
- 直接取自專案的程式碼片段（含逐行註解）

**必須涵蓋的概念**（最低要求——若專案有使用其他概念，請一併補充）：

| 概念 | 說明重點 |
|---|---|
| 標頭檔與實作檔 | 為何要將 `.h` 與 `.cpp` 分開（與 Java 的差異） |
| 命名空間 | 如何避免名稱衝突 |
| 類別、繼承、多型 | 與 Java 類別模型的比較 |
| 建構子與解構子 | 生命週期管理；RAII 模式 |
| 指標（`*`）與參考（`&`） | 語意差異，各自的使用時機 |
| 智慧指標 | `unique_ptr`、`shared_ptr`、`weak_ptr` 及其存在原因 |
| 記憶體：堆疊 vs. 堆積 | 變數存放的位置與負責清理的對象 |
| 樣板 | 泛型程式設計，與 Java 泛型的比較 |
| 例外處理 | C++ 中的 `try`／`catch`／`throw` |
| RAII | 將資源生命週期綁定於物件生命週期的慣用法 |

**第一階段執行提示詞：**
> 請查看 `include/` 與 `source/` 資料夾，並向我解說這些程式碼，因為我完全是 C++ 新手。請涵蓋所有重要的 C++ 概念，並將結果寫入 `explanation.md` 檔案中。越詳細越好。

第一階段完成後，請在繼續之前檢查 `explanation.md` 是否完整。

---

## 第二階段 — 深化內容與補充缺漏

**目標：** 找出第一階段遺漏的部分，並讓每個項目更加豐富。

1. 重新全面掃描程式碼庫（必要時使用目錄樹）
2. 閱讀現有的 `explanation.md`
3. 針對每個項目，確認專案中是否還有相關程式碼尚未提及
4. 針對每個 C++ 概念，若以下四項尚未出現，請全部補充：
   - **白話類比**（假設你在向 5 歲小孩解釋）
   - **Java 比較**（在 Java 中會如何實作？）
   - **專案中的實際程式碼片段**
   - **使用時機與原因**

**第二階段執行提示詞：**
> 請先分析整個專案程式碼庫，然後閱讀 `explanation.md` 檔案。嘗試補充更多細節，讓 `explanation.md` 更加完整清晰。請記住一件重要的事：解釋所有重要概念，並把我當成一個完全不懂 C++ 的五歲小孩來對待。

第二階段完成後，確認每個項目都有足夠深度，且沒有遺漏任何原始檔。

---

## 第三階段 — 回頭補強項目 #1–#20

**目標：** 第二階段很可能發現了與早期項目相關的新程式碼，請回頭補充。

- 閱讀 `explanation.md` 中的項目 #1 到 #20
- 對每個項目確認：第二階段是否發現了與此項目相關、但尚未提及的程式碼？
- 若有，請將新的程式碼片段與說明加入該項目中
- **不要重新編號或調整結構**——只在現有項目內新增內容

**第三階段執行提示詞：**
> 別忘了也要修改 `explanation.md` 中的項目 #1 到 #20，因為第二階段也發現了與這些項目相關的新程式碼。

第三階段完成後，整份文件應感覺完整且前後一致。

---

## 第四階段 — 翻譯為繁體中文

**目標：** 讓說明文件對中文讀者更加易懂。

將所有說明文字翻譯為繁體中文。以下內容**保留英文，不得翻譯**：
- 程式碼區塊內的所有內容
- C++ 標準函式庫名稱（例如 `std::vector`、`std::unique_ptr`）
- 函式名稱、變數名稱、類別名稱
- C++ 技術術語可保留英文，但需在括號內附上中文說明，例如：`RAII（資源取得即初始化）`

風格：口語化、易於閱讀——讀者有技術背景，但從未接觸過 C++。

**第四階段執行提示詞：**
> 請將 `explanation.md` 檔案的語言從英文改為繁體中文。

## 輸出規格

| 欄位 | 值 |
|---|---|
| 檔案 | `explanation.md`（位於專案根目錄） |
| 語言 | 繁體中文（程式碼與術語保留英文） |
| 結構 | 編號項目：項目 #1、項目 #2…… |
| 深度 | 適合有 Java 背景的完全 C++ 初學者 |
| 程式碼註解 | 逐行或逐區塊說明 |

---

## 注意事項

- 每個階段結束後，**請暫停並閱讀輸出結果**，再繼續進行下一階段。品質比速度更重要。
- 若專案結構特殊（無 `include/` 資料夾、單一大型檔案等），請相應調整掃描策略，並告知使用者你所發現的情況。
- 若某個概念出現在多個檔案中，請在主要說明中選用最清晰的範例，並以交叉參考的方式提及其他出現位置。
- 第三階段**只新增內容**——保留所有現有的項目編號與結構不變。
